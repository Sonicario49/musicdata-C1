## Notes

- **Séparation E/T** : chaque script d'extraction ne sauvegarde que la donnée brute, dans le format natif de sa source (JSON, HTML, CSV natif, Parquet natif). Aucun renommage vers le schéma commun ni conversion d'unité n'a lieu à l'extraction : c'est `aggregation/aggregate.py` (C3) qui parse, nettoie et harmonise les 5 sources vers le schéma cible.
- Schéma cible (après agrégation) : `titre, artiste, genre, durée (secondes), date de sortie, popularité, source`.
- Chaque champ n'est pas garanti disponible sur chaque source (ex : pas de date de sortie via le scraping ni le fichier CSV Kaggle ; pas de genre via la source Big Data ; pas de popularité via MusicBrainz) , l'agrégation (C3) gère ces absences (valeurs vides ou `"inconnu"`).
- Les popularités ne sont **pas comparables entre elles** d'une source à l'autre (rank Deezer, streams/jour Spotify via kworb, score interne Spotify via les datasets Kaggle/Hugging Face) : elles restent des indicateurs propres à chaque plateforme, pas une métrique unifiée.
- Tous les scripts d'extraction respectent : point d'entrée `if __name__ == "__main__":`, gestion des erreurs (try/except), logs explicites.
