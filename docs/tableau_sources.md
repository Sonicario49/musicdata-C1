## Notes

- Toutes les sources convergent vers le même schéma cible : `titre, artiste, genre, durée (secondes), date de sortie, popularité, source`.
- Chaque champ n'est pas garanti disponible sur chaque source (ex : pas de date de sortie via le scraping ni le fichier CSV Kaggle ; pas de popularité via MusicBrainz) , l'agrégation (C3) devra gérer ces absences (valeurs vides ou `"inconnu"`).
- Les popularités ne sont **pas comparables entre elles** d'une source à l'autre (rank Deezer, streams/jour Spotify via kworb, score interne Spotify via le dataset Kaggle) : elles restent des indicateurs propres à chaque plateforme, pas une métrique unifiée.
- Tous les scripts d'extraction sauvegardent leur résultat normalisé dans `data/raw/`, et respectent : point d'entrée `if __name__ == "__main__":`, gestion des erreurs (try/except), logs explicites.
